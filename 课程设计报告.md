# 网上订餐管理系统的设计与实现

## 一、课题分析

### 1.1 课题背景

随着互联网技术的飞速发展和人们生活节奏的加快，网上订餐已成为日常生活中不可或缺的一部分。传统的电话订餐方式存在信息不透明、效率低下、容易出错等弊端，而网上订餐系统能够提供直观的菜品展示、便捷的下单流程和实时的订单追踪，极大提升了用户的订餐体验和商家的管理效率。

### 1.2 可行性分析

| 维度 | 分析 |
|------|------|
| **技术可行性** | 系统采用 Go 语言 + Gin 框架作为后端，SQLite 作为数据库，HTML/CSS/JavaScript 作为前端，技术栈成熟稳定，开发工具和文档资源丰富 |
| **经济可行性** | 全部采用开源技术，SQLite 无需独立数据库服务器，部署成本低；Go 语言编译为单一可执行文件，运维简便 |
| **操作可行性** | 前端界面采用 Google Material Design 风格，交互直观友好；后台管理提供完整的数据统计和 CRUD 操作，易于上手 |
| **社会可行性** | 网上订餐已成为主流消费方式，系统符合当前市场需求，具有良好的实用价值 |

### 1.3 研究目标

设计并实现一个功能完善的网上订餐管理系统，包含前台用户订餐和后台管理两大模块，实现从菜品浏览、购物车管理、在线下单到订单追踪的完整业务闭环，以及后台的分类管理、菜品管理、订单管理和用户管理等功能。

## 三、总体设计

### 3.1 系统架构

系统采用 **B/S 架构**（Browser/Server），基于前后端分离模式设计：

```

│                       客户端（浏览器）                      │
│  ┌──────────────────┐          ┌──────────────────┐      │
│  │   前台页面 (index)  │          │  后台页面 (admin)   │      │
│  │   HTML/CSS/JS     │          │   HTML/CSS/JS     │      │
│  └────────┬─────────┘          └────────┬─────────┘      │
└───────────┼──────────────────────────────┼───────────────┘
            │         HTTP/HTTPS            │
            ▼                              ▼
┌──────────────────────────────────────────────────────────┐
│                    Go 后端服务 (Gin)                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 路由层    │→ │ 中间件层   │→ │ 控制器层   │→ │ 模型层   │ │
│  │ routes   │  │ middleware│  │controllers│  │ models  │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
└──────────────────────────┬───────────────────────────────┘
                           │
                           ▼
                  ┌─────────────────┐
                  │  SQLite 数据库    │
                  │  ordering.db    │
                  └─────────────────┘
```

### 3.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 后端语言 | Go 1.25 | 高性能、编译型语言，天生支持并发 |
| Web 框架 | Gin v1.12 | 轻量级高性能 HTTP 框架 |
| ORM | GORM v1.31 | Go 语言最流行的 ORM 框架 |
| 数据库 | SQLite | 嵌入式数据库，无需独立部署 |

## 六、系统实现

### 6.1 数据库连接与模型定义

使用 GORM 连接 SQLite 数据库，通过 `AutoMigrate` 自动建表：

```go
func ConnectDB(dbname string) {
    db, err := gorm.Open(sqlite.Open(dbname), &gorm.Config{})
    if err != nil {
        panic("failed to connect database")
    }
    DB = db
    DB.AutoMigrate(&User{}, &Category{}, &Dish{}, &CartItem{}, &Order{}, &OrderItem{})
}
```

模型定义示例（订单模型）：

```go
type Order struct {
    ID      uint        `gorm:"primaryKey" json:"id"`
    UserID  uint        `gorm:"not null;index" json:"user_id"`
    Total   float64     `gorm:"not null" json:"total"`
    Status  string      `gorm:"size:20;default:pending" json:"status"`
    Address string      `gorm:"size:255" json:"address"`
    Phone   string      `gorm:"size:20" json:"phone"`
    Remark  string      `gorm:"size:500" json:"remark"`
    Items   []OrderItem `gorm:"foreignKey:OrderID" json:"items,omitempty"`
    User    User        `gorm:"foreignKey:UserID" json:"user,omitempty"`
}
```

### 6.2 认证与权限控制

#### JWT Token 生成

```go
func GenerateToken(userID uint, username, role string) (string, error) {
    claims := jwt.MapClaims{
        "user_id":  userID,
        "username": username,
        "role":     role,
        "exp":      time.Now().Add(24 * time.Hour).Unix(),
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString([]byte(config.AppConfig.JWTSecret))
}
```

#### 认证中间件

系统采用双层中间件实现权限控制：

- **AuthMiddleware**：解析 JWT Token，提取用户信息存入上下文
- **AdminMiddleware**：校验用户角色是否为管理员

```go
admin := api.Group("/admin")
admin.Use(middleware.AuthMiddleware(), middleware.AdminMiddleware())
{
    admin.GET("/users", controllers.ListUsers)
    admin.POST("/categories", controllers.CreateCategory)
    // ...
}
```

### 6.3 核心功能实现

#### 6.3.1 用户注册

- 校验用户名唯一性
- bcrypt 加密存储密码
- 自动生成 JWT Token 返回

#### 6.3.2 购物车管理

- 添加菜品时，若已存在则累加数量，否则新建记录
- 修改数量时校验所属用户和最小值
- 下单成功后自动清空购物车

## 八、总结与展望

### 8.1 系统总结

本系统基于 Go 语言 + Gin 框架 + SQLite 数据库，成功实现了网上订餐管理系统的设计与开发。系统具有以下特点：

1. **功能完整**：覆盖了从用户注册、菜品浏览、购物车管理、在线下单到后台数据管理的完整业务流程
2. **架构清晰**：采用 MVC 分层架构，路由、中间件、控制器、模型职责分明
3. **安全可靠**：JWT 认证 + bcrypt 加密 + 权限中间件，保障系统安全
4. **界面美观**：采用 Google Material Design 设计语言，界面简洁现代
5. **部署便捷**：Go 编译为单一可执行文件，SQLite 嵌入式数据库，零依赖部署

### 8.2 不足与展望

| 方面 | 当前状态 | 改进方向 |
|------|---------|---------|
| 数据库 | SQLite 单文件 | 可迁移至 MySQL/PostgreSQL 支持高并发 |
| 支付 | 无在线支付 | 接入支付宝/微信支付 API |
| 实时通知 | 无 | 引入 WebSocket 实现订单状态实时推送 |
| 前端框架 | 原生 JS | 可迁移至 Vue/React 提升交互体验 |
| 图片存储 | 本地文件系统 | 可迁移至 OSS 云存储 |
| 数据统计 | 简单数字统计 | 增加图表可视化、趋势分析 |
| 多商家 | 单商家模式 | 扩展为多商家入驻平台 |
